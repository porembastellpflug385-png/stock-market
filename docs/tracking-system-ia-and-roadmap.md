# Tracking System IA And Roadmap

## 1. 页面信息架构

### 1.1 首页 Dashboard

目标：
给出全局总览

模块：

1. 关注池概览
2. 今日新增信号
3. 各模拟盘表现
4. AI 结论命中率
5. 今日风险预警

### 1.2 关注池页面

目标：
管理长期观察标的

模块：

1. 标的列表
2. 标签筛选
3. 风险筛选
4. 最新 AI 结论
5. 最新策略动作

### 1.3 标的详情页

目标：
查看单标的完整跟踪链路

模块：

1. 价格与指标图
2. 历史 AI 报告时间轴
3. 支撑/阻力与信号变化
4. 模拟盘策略动作记录
5. AI 验证结果

### 1.4 模拟盘总览页

目标：
比较不同策略代理人

模块：

1. 策略净值曲线
2. 收益率、回撤、胜率
3. 当前持仓
4. 最新调仓记录

### 1.5 单策略详情页

目标：
拆解某个策略代理人的执行逻辑和表现

模块：

1. 策略规则摘要
2. 当前仓位
3. 历史交易记录
4. 收益归因
5. 与 AI 结论一致性分析

### 1.6 验证中心

目标：
回答“AI 到底准不准”

模块：

1. 方向命中率
2. 置信度有效性
3. 分市场命中率
4. 分资产类别命中率
5. 分策略适配度
6. 失败案例复盘

### 1.7 报告中心

目标：
汇总自动产出的日报、周报、月报

模块：

1. 日报列表
2. 周报列表
3. 月报列表
4. 验证报告列表
5. PDF / Markdown / HTML 下载

## 2. MVP 页面范围

第一版建议只做：

1. 首页 Dashboard
2. 关注池页面
3. 标的详情页
4. 模拟盘总览页
5. 验证中心简版
6. 报告中心简版

## 3. 关键用户故事

### 3.1 作为研究用户

我想把贵州茅台、特斯拉、比特币加入关注池，这样我可以持续跟踪信号和结论变化。

### 3.2 作为策略观察者

我想看“成长趋势策略”和“量化轮动策略”在同一组标的上的表现差异，这样我能判断哪种风格更适合当前市场。

### 3.3 作为验证用户

我想知道过去 20 天里 GPT 对比特币和特斯拉的判断到底准不准，这样我能决定以后要不要继续参考 AI 结论。

### 3.4 作为复盘用户

我想看周报和月报，这样我能知道错误最多的是哪类判断、最有效的是哪类策略。

## 4. 开发路线图

### Phase 1: MVP 闭环

目标：
先把“关注池 + AI 日报 + 单策略模拟盘 + 基础验证”跑起来

内容：

1. 关注池管理
2. 单标的历史分析记录
3. 2 个策略代理人
4. 模拟盘交易记录
5. 日报 / 周报
6. 方向命中率验证

预计产出：

1. 可持续运行的研究闭环
2. 能回看 AI 报告
3. 能量化“准不准”

### Phase 2: 多策略与归因

目标：
把验证体系做深

内容：

1. 增加到 4-5 个策略代理人
2. 策略表现归因
3. 置信度与命中率关系分析
4. 分市场与分资产类别验证
5. 月报

### Phase 3: 自动化与智能提醒

目标：
提高可用性和持续跟踪能力

内容：

1. 自动提醒
2. 异常信号推送
3. 自动产出验证报告
4. 用户自定义策略参数

## 5. 推荐排期

### Sprint 1

1. 数据模型
2. 关注池
3. AI 报告存档
4. 标的详情页时间轴

### Sprint 2

1. 2 个策略代理人
2. 模拟盘组合
3. 交易流水
4. 净值曲线

### Sprint 3

1. 验证引擎
2. 日报 / 周报
3. 验证中心
4. 命中率看板

### Sprint 4

1. 月报
2. 归因分析
3. 报告导出
4. 运营和监控补强

## 6. 我建议的下一步

如果马上进入开发，建议优先顺序是：

1. 数据库设计落地
2. 关注池与 AI 报告存档
3. 双策略模拟盘
4. 验证中心 MVP

也就是说，先把“记录”和“回看”能力做扎实，再做复杂策略。
